## Jwt 相关

### 刷新 Token

| 业务动作 | Controller 方法 | 路径                        | 入参 | 出参类    | 状态 |
| -------- | --------------- | --------------------------- | ---- | --------- | ---- |
| 刷新令牌 | `refreshToken`  | `POST /common/auth/refresh` | 无   | `LoginVO` | ✅   |

提供一个刷新 Token 接口，返回时除了新 Token，还要返回用户最新信息来刷新。以此实现无感刷新：

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库

    Note over Client,Auth: 请求已通过 JWT Filter 校验，Spring Security 上下文已包含 userId

    Client->>Auth: POST /common/auth/refresh<br/>Header: Authorization: Bearer {current_jwt}
    Auth->>Auth: 从 Spring Security 上下文获取 userId
    Auth->>DB: 获取用户最新数据（昵称、头像、状态等）
    DB-->>Auth: 返回用户数据
    alt 用户已封禁
        Auth-->>Client: 403 Forbidden<br/>{code: "ACCOUNT_DISABLED", msg: "账号已被封禁"}
    else 用户正常
        Auth->>Auth: 生成新的 JWT
        Auth-->>Client: 200 OK<br/>LoginVO（包含最新用户数据）
        Client->>Client: 保存新 JWT 并更新本地用户信息
    end
```

### 账号注册

必须携带 username, email/phone（二选一），其它非敏感字段可选：

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库

    Client->>Auth: POST /common/auth/register<br/>UserRegisterDTO
    Auth->>Auth: 校验验证码（手机号/邮箱与验证码匹配，未过期）
    alt 验证码无效或错误
        Auth-->>Client: 400 Bad Request<br/>{msg: "验证码错误或已过期"}
    else 验证码正确
        Auth->>DB: 检查手机号/邮箱是否已注册
        alt 手机号/邮箱已存在
            Auth-->>Client: 409 Conflict<br/>{msg: "该账号已注册"}
        else 手机号/邮箱未注册
            Auth->>DB: 检查用户名是否已存在
            alt 用户名已存在
                Auth-->>Client: 409 Conflict<br/>{msg: "用户名已存在"}
            else 用户名未存在
                Auth->>DB: 创建新用户记录
                Auth-->>Client: 200 OK
            end
        end
    end
```

### 获取播放签名

签名就是临时 URL：

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库
    participant Minio as MinIO

    Client->>Auth: GET /common/file/sign?audioId=1024
    Auth->>Auth: 从 Spring Security 上下文获取 userId（可选）
    Auth->>DB: 查询音频信息（trialDuration, isPaid, 试听文件是否存在）
    DB-->>Auth: 返回音频信息

    alt 用户已购买 或 音频免费
        Auth->>Minio: 生成完整音频文件的预签名URL
        Minio-->>Auth: 返回完整音频URL
        Auth-->>Client: 200 OK<br/>{完整音频URL}
    else 未购买且音频付费
        alt trialDuration > 0 且试听文件已生成
            Auth->>Minio: 生成试听片段文件的预签名URL
            Minio-->>Auth: 返回试听URL
            Auth-->>Client: 200 OK<br/>{试听URL}
        else 试听未就绪或无试听
            Auth-->>Client: 403 Forbidden<br/>{msg: "请购买后收听完整版"}
        end
    end
```

> [!NOTE] 观看次数的实现
> `playCount`（播放次数）要在获取播放签名接口中累加。每次前端调用 `/common/file/sign` 获取播放 URL 时，后端判断为一次有效播放请求，将对应音频的 `play_count` 字段 +1。
> 
> **注意**：为防止恶意刷量，需要增加限流（同一用户 对同一音频短时间内多次请求只计一次），用 Redis 记录 `user:play:counted:{userId}:{audioId}`，过期时间设为 1 小时。

### 收藏/取消收藏

```json
// FavoriteActionDTO：收藏/取消收藏请求参数
{
  "audioId": 1024,           // 音频ID（必填）
  "folderId": 1,             // 收藏夹ID（可选，不传则收藏至默认收藏夹）
  "action": "COLLECT"        // COLLECT（收藏）/ UNCOLLECT（取消收藏）
}
```

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库

    Client->>Auth: POST /user/audio/action<br/>FavoriteActionDTO
    Auth->>Auth: 从 Spring Security 上下文获取 userId

    alt action = COLLECT
        Auth->>DB: 检查音频是否已在目标收藏夹中
        alt 已存在
            Auth-->>Client: 409 Conflict<br/>{msg: "音频已收藏"}
        else 未存在
            Auth->>DB: 新增收藏记录
            Auth-->>Client: 200 OK
        end
    else action = UNCOLLECT
        Auth->>DB: 检查收藏记录是否存在
        alt 不存在
            Auth-->>Client: 404 Not Found<br/>{msg: "收藏记录不存在"}
        else 存在
            Auth->>DB: 删除收藏记录
            Auth-->>Client: 200 OK
        end
    end
```

## 创作者申请

支持普通用户申请为创作者。前缀为 `/user`：

| 业务动作     | Controller 方法  | 路径                    | 方法 | 入参              | 出参类           | 状态 |
| ------------ | ---------------- | ----------------------- | ---- | ----------------- | ---------------- | ---- |

| 查询申请状态 | `getApplyStatus` | `/creator/apply/status` | GET  | 无                | `CreatorApplyVO` |      |

### 提交申请

```json
// CreatorApplyDTO：创作者申请参数
{
  "realName": "张三",           // 真实姓名（必填）
  "phone": "13800000000",      // 联系电话（必填）
  "intro": "资深后端工程师，擅长技术分享",  // 个人简介（可选）
	...
  "attachment": "https://..."  // 证明材料链接（可选）
}
```

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库

    Client->>Auth: POST /user/creator/apply<br/>CreatorApplyDTO
    Auth->>Auth: 从 Spring Security 上下文获取 userId
    Auth->>DB: 检查用户是否已是创作者
    alt 已是创作者
        Auth-->>Client: 400 Bad Request<br/>{msg: "您已是创作者"}
    else 非创作者
        Auth->>DB: 检查是否有进行中的申请

        else 无申请
            Auth->>DB: 创建创作者申请记录（状态：待审核）
            Auth-->>Client: 200 OK
        end
    end
```
