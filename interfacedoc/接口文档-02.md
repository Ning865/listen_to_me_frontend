### 上传音频

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant Minio as MinIO
    participant Redis as Redis

    Client->>Auth: POST /creator/audio/upload<br/>MultipartFile
    Auth->>Auth: 从 Spring Security 上下文获取当前用户权限
    Auth->>Auth: 校验是否有 'audio:upload' 权限
    alt 无权限
        Auth-->>Client: 403 Forbidden<br/>{msg: "无权上传音频相关文件"}

        Auth->>Redis: RedisUtils.set(RedisKey.UPLOAD_TEMP, tempUrl, objectKey)

        Auth-->>Client: 200 OK<br/>{tempUrl}
        Note over Client: 客户端保存 tempUrl 用于预览和后续发布
    end
```

### 创建订单

用户使用虚拟币购买音频，直接扣减余额并生成订单。处于简单考虑，目前只支持使用虚拟币创建：

```json
// OrderVO 出参
{
  "orderSn": "ORD202603280001",
  "audioId": 1024,
  "payAmount": 199,
  "balanceAfter": 2801,
  "status": "SUCCESS"
}
```

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库

    Client->>Auth: POST /user/order/1024
    Auth->>Auth: 从 Spring Security 上下文获取 userId
    Auth->>DB: 查询音频信息（价格、是否付费、状态）
    alt 音频不存在或已下架
        Auth-->>Client: 404 Not Found
    else 音频免费
        Auth-->>Client: 400 Bad Request<br/>{msg: "该音频免费，无需购买"}
    else 校验通过
        Auth->>DB: 查询用户是否已购买该音频（幂等性）
        alt 已购买
            Auth-->>Client: 409 Conflict<br/>{msg: "您已购买过该音频"}
        else 未购买
            Auth->>DB: **开启事务（@Transactional）**<br/>1. 乐观锁扣减余额：<br/>   UPDATE user_balance SET balance = balance - ? <br/>   WHERE user_id = ? AND balance >= ?<br/>2. 若更新行数 == 0 → 余额不足，回滚事务，返回失败<br/>3. 创建订单记录（状态：成功）<br/>4. 记录虚拟币流水（支出）<br/>5. 添加用户购买权限记录<br/>6. 提交事务
            alt 扣减失败（影响行数 == 0）
                Auth-->>Client: 200 OK<br/>{status: "FAILED_INSUFFICIENT", balanceAfter: 当前余额（可从数据库查询）}
            else 扣减成功
                Auth-->>Client: 200 OK<br/>{status: "SUCCESS", balanceAfter: 更新后的余额}
            end
        end
    end
```


## AI 增强

前缀为 `/creator`（申请任务）和 `/common`（查询任务状态）：

|业务动作|Controller 方法|路径|方法|入参|出参类|状态|
|---|---|---|---|---|---|---|
|申请 AI 转写|`saveAiTranscript`|`/ai/transcript`|POST|`AudioIdDTO`|`AiTaskVO`||
|申请 AI 摘要|`saveAiNote`|`/ai/note`|POST|`AudioIdDTO`|`AiTaskVO`||
|查询 AI 任务状态|`getAiTask`|`/ai/task/{taskId}`|GET|路径参数 `taskId`|`AiTaskVO`|

### 请 AI 转写

```json
// AiTaskVO 出参
{
  "taskId": "task_991",
  "status": "PENDING"        // PENDING / PROCESSING / SUCCESS / FAILED
}
```

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Auth as 认证服务
    participant DB as 数据库
    participant Queue as 消息队列

    Client->>Auth: POST /creator/ai/transcript<br/>AudioIdDTO
    Auth->>Auth: 从 Spring Security 上下文获取 userId
    Auth->>DB: 校验音频是否存在且属于当前用户
    alt 音频不存在或无权限
        Auth-->>Client: 404 Not Found
    else 存在
        Auth->>DB: 创建 AI 任务记录（状态：PENDING）
        DB-->>Auth: 返回 taskId
        Auth->>Queue: 发送转写任务（taskId, audioId）
        Auth-->>Client: 200 OK<br/>AiTaskVO
    end
```

异步调用如下：

```mermaid
sequenceDiagram
    participant Queue as 消息队列
    participant Worker as 转写Worker
    participant Minio as MinIO
    participant AI as AI服务
    participant DB as 数据库

    Queue-->>Worker: 消费转写任务（taskId, audioId）
    Worker->>DB: 更新任务状态为 PROCESSING
    Worker->>Minio: 下载音频文件
    Minio-->>Worker: 返回音频数据
    Worker->>AI: 调用语音转写接口
    AI-->>Worker: 返回转写结果（segments）
    alt 调用成功
        Worker->>DB: 更新任务状态为 SUCCESS，保存转写结果
    else 调用失败
        Worker->>DB: 更新任务状态为 FAILED，记录失败原因
    end
```
